# Универсальный профиль пользователя v1

Статус: **утверждённая техническая основа для следующего этапа разработки**
Вторая страна MVP: **Уругвай**
Профиль используется одинаково для Испании, Уругвая и следующих стран.

## 1. Назначение

`user-profile-v1` — единый контракт ответов пользователя. Он заменяет испанскую техническую форму, в которой общие вопросы были смешаны с требованиями отдельных испанских вариантов легализации.

Профиль должен отвечать только на вопросы о пользователе:

- гражданства;
- текущее место проживания и понятный фактический статус;
- допустимые места подачи;
- семья и дети;
- источник, сумма, валюта и подтверждаемость дохода;
- долгосрочная цель;
- бюджет и условия жизни;
- животные;
- особые обстоятельства.

В профиль **не входят**:

- ручной курс валют;
- названия конкретных программ;
- Seguridad Social;
- требования конкретного консульства;
- специфические документы Испании или Уругвая;
- вывод «какой вариант легализации выбран».

Эти данные принадлежат пакету страны, расчётному контексту или уточняющим вопросам после первого расчёта.

## 2. Главные решения

### 2.1 Туристическая виза — отдельный понятный статус

Пользователь не отвечает на вопрос «являетесь ли вы законным резидентом». Он выбирает фактический статус:

- гражданство;
- ПМЖ;
- временный ВНЖ;
- рабочая или семейная виза;
- студенческий статус;
- туристическая виза или безвизовое пребывание;
- другой законный статус;
- законного статуса нет.

Движок сам определяет, достаточно ли этого статуса для подачи по конкретному варианту легализации.

### 2.2 Деньги всегда хранятся как сумма + валюта

Пример:

```json
{
  "amount": 3000,
  "currency": "USD"
}
```

Пользователь не вводит курс. Курс передаётся движку отдельным расчётным контекстом, где обязательно фиксируются источник и дата.

### 2.3 Состав семьи хранится без потери детей

Отдельно хранятся:

- количество взрослых;
- наличие партнёра;
- тип отношений;
- массив детей;
- возраст каждого ребёнка;
- необходимость школы.

Экран результата должен выводить, например: `2 взрослых, 1 ребёнок`, а не заменять ребёнка вторым взрослым.

### 2.4 Основной источник дохода не выбирает вариант легализации

`income.primary.type` описывает факт о доходе. Он используется для проверки совместимости со всеми вариантами легализации, но не назначает заранее «целевой» вариант.

### 2.5 Язык относится к долгосрочной цели

Поле называется `goal.language_exam_readiness` и отвечает на вопрос:

> Готовы ли вы сдавать экзамен по местному языку для ПМЖ или гражданства?

Оно не должно блокировать первоначальный ВНЖ, если конкретный вариант не требует языка на первом этапе.

### 2.6 Бюджет — это максимальные расходы семьи

`preferences.monthly_budget` означает максимальные ежемесячные расходы на:

- аренду;
- продукты;
- транспорт;
- медицину;
- обычные бытовые расходы.

Школа считается отдельно. `null` означает, что бюджет пока не определён.

### 2.7 Медицина и порода собаки — практические уточнения

Они не меняют юридический статус ВНЖ сами по себе.

- медицинский блок необязателен;
- порода собаки нужна только для проверки официальных ограничений на ввоз, содержание и перевозку;
- отсутствие ответа не делает страну юридически неподходящей.

## 3. Структура профиля

### 3.1 `citizenships`

Массив содержит единственное значение `RU`. Текущая версия продукта предназначена только для граждан РФ: другие и дополнительные гражданства не запрашиваются и не участвуют в расчёте.

### 3.2 `residence`

```json
{
  "current_country": "PH",
  "current_status": "TOURIST_OR_VISA_FREE"
}
```

### 3.3 `application_preferences`

Допустимые способы подачи:

- `CURRENT_COUNTRY`;
- `RUSSIA`;
- `IN_COUNTRY_AFTER_ENTRY`;
- `ANY`.

### 3.4 `family`

```json
{
  "adults_count": 2,
  "partner_included": true,
  "relationship_type": "MARRIAGE",
  "children": [{ "age_years": 13 }],
  "school_needed": true
}
```

### 3.5 `lgbt`

Содержит включение отдельного трека и согласие на использование этих сведений в персональном расчёте. Подробности признания семьи и безопасности могут уточняться после первого расчёта.

### 3.6 `income`

Основной доход, дополнительные источники и доход партнёра хранятся отдельно.

У каждого источника есть:

- владелец;
- тип;
- страна источника;
- страна банковского счёта;
- подтверждаемая сумма и валюта;
- уровень документального подтверждения;
- необязательные сведения об истории и стабильности.

### 3.7 `goal`

- необходимый долгосрочный результат;
- готовность фактически жить в стране;
- готовность к языковому экзамену;
- важность сохранения гражданства РФ.

### 3.8 `preferences`

- максимальный ежемесячный бюджет семьи;
- размер города;
- один или несколько подходящих вариантов климата.

### 3.9 `pets`

Массив типов животных. Сведения о породе собаки могут быть `null`, если пользователь не знает породу или не хочет её указывать.

### 3.10 `special_circumstances`

Судимость, депортация, запрет на въезд и серьёзное нарушение визового режима не исключают страну автоматически. Они переводят затронутые варианты в `INDIVIDUAL_REVIEW_REQUIRED`.

### 3.11 `route_specific_answers`

Отдельное пространство для уточнений, которые показываются только после первого расчёта. Ключ — `route_id`.

Пример:

```json
{
  "ES_DNV": {
    "social_security_plan": "UNKNOWN"
  }
}
```

Этот объект не является частью основной анкеты и валидируется отдельной схемой конкретного варианта легализации.

## 4. Строгий контракт

Машиночитаемая схема:

`data/schemas/user-profile-v1.schema.json`

Она соответствует JSON Schema Draft 2020-12.

## 5. Правила пустых значений

- отсутствующий ответ не превращается в `false` или `0`;
- неизвестный бюджет хранится как `null`, а не как нулевые расходы;
- отсутствие партнёра — `partner_included: false` и `relationship_type: null`;
- отсутствие детей — пустой массив;
- отсутствие дохода партнёра — `has_income: false` и пустой массив источников;
- неизвестная порода — `breed: null`;
- ручной курс валют нигде не хранится.

## 6. Граница между общей анкетой и уточнениями

Общая анкета должна завершаться предварительным сравнением стран. После него система показывает только те уточняющие вопросы, которые реально способны изменить результат конкретного варианта.

Примеры:

- Испания DNV: способ подтверждения социального страхования;
- Уругвай: форма доказательства достаточных средств;
- конкретный маршрут: срок договора или возраст бизнеса;
- конкретная страна: согласие второго родителя на переезд ребёнка.

Ответ `Не знаю` должен быть допустим. Он приводит к `PRELIMINARY_SUITABLE`, `INSUFFICIENT_COUNTRY_DATA` или `INDIVIDUAL_REVIEW_REQUIRED`, а не автоматически к `UNSUITABLE`.

## 7. Критерии готовности профиля

Профиль v1 считается готовым, когда:

1. все 29 вопросов основной анкеты имеют однозначный путь в JSON;
2. ни одно поле не содержит названия страны или программы;
3. сумма и валюта хранятся вместе;
4. дети не сводятся к количеству взрослых;
5. туристический статус отличим от резидентства;
6. основной источник дохода не назначает победивший вариант легализации;
7. медицинские и бытовые предпочтения не изменяют юридический статус маршрута;
8. один и тот же профиль можно без преобразования передать в расчёт Испании и Уругвая.
