# Immigration Country Matcher

Персональный матчер стран и иммиграционных программ для граждан РФ. Одна универсальная анкета формирует профиль `user-profile-v1`, после чего движок проверяет программы каждой подключённой страны и отдельно оценивает практические условия жизни.

## Рабочая версия

Публичная страница:

- `https://sankhipkate.github.io/immigration-country-matcher/matcher/`

Версия интерфейса: **0.12.3**.

Корневой адрес проекта и прежний адрес `/pilot/` не показывают отдельные интерфейсы и перенаправляют пользователя на `/matcher/`.

## Что уже работает

- универсальная анкета для граждан РФ;
- прямое формирование профиля `user-profile-v1`;
- сравнение Испании и Уругвая по одной анкете;
- проверка всех доступных иммиграционных программ;
- отдельная оценка юридической доступности и бытового бюджета; если бюджет не указан, он автоматически приравнивается к общему регулярному доходу;
- расчёт подтверждаемого дохода в разных валютах;
- семейные требования, партнёр, дети и возраст каждого ребёнка;
- практические сведения о городах, климате, школе и животных; строка о школе показывается только для платной международной школы;
- необязательный раздел о ЛГБТ-правах, семейной иммиграции, безопасности и международной защите;
- сохранение и очистка черновика в браузере;
- официальные источники и дата расчёта в результате.
- понятные инструкции о месте и способе подачи по каждому маршруту;
- срок и ограничения пути к гражданству внутри каждой карточки маршрута;
- семейный маршрут Уругвая как будущая возможность для заявителя без текущего партнёра;
- постоянная резиденция Уругвая считается подходящей при подтверждаемом доходе выше 650 USD;
- список стран остаётся слева на компьютере, а на телефоне превращается в горизонтальные вкладки;
- версия JS, CSS и страновых JSON закреплена в URL, чтобы браузер не показывал старые файлы после публикации.

## Доход в анкете

Для каждого источника пользователь указывает общий регулярный доход и затем выбирает:

- весь доход можно подтвердить;
- можно подтвердить только часть;
- пока доход подтвердить нельзя.

Поле отдельной подтверждаемой суммы появляется только при выборе «Только часть». Пустая подтверждаемая сумма блокирует переход именно на шаге дохода, а не в конце анкеты.

## Архитектура

- `matcher/` — публичная универсальная анкета и результаты;
- `matcher/profile.js` — формирование и проверка `user-profile-v1`;
- `js/engine/` — универсальный движок расчёта;
- `js/countries/` — страновые адаптеры;
- `data/spain-research-v2.2.json` — исследовательский пакет Испании;
- `data/uruguay-research-v2.2.json` — исследовательский пакет Уругвая;
- `data/research-package-v2.2.schema.json` — схема исследовательского пакета;
- `data/schemas/user-profile-v1.schema.json` — схема профиля пользователя;
- `docs/architecture/` — архитектурные решения и карта анкеты;
- `tests/` — автоматические проверки движка, профиля, страновых пакетов и интерфейса.

Используются шесть канонических статусов:

- `SUITABLE`;
- `SUITABLE_WITH_CONDITIONS`;
- `PRELIMINARY_SUITABLE`;
- `INSUFFICIENT_COUNTRY_DATA`;
- `INDIVIDUAL_REVIEW_REQUIRED`;
- `UNSUITABLE`.

Недостающий ответ пользователя и недостающие данные исследования страны не смешиваются. Неизвестные сведения не превращаются автоматически в отказ.
